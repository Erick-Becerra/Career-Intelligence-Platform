Quiero que actúes como el responsable de documentación del proyecto.

Genera un documento llamado **"Estado del Proyecto"** que sirva para continuar el desarrollo en un nuevo chat sin perder contexto.

El documento debe estar escrito en Markdown y contener las siguientes secciones, siempre en este orden:

# Estado del Proyecto

## 1. Resumen del proyecto

Explica en pocas líneas cuál es el objetivo general del proyecto.

## 2. Objetivo final

Describe claramente qué se espera lograr cuando el proyecto esté terminado.

## 3. Arquitectura actual

Explica cómo está organizada actualmente la aplicación, incluyendo la estructura de carpetas, los componentes existentes y cómo interactúan entre sí.

## 4. Agentes definidos

Lista todos los agentes planeados hasta el momento e indica el propósito de cada uno.

## 5. Tecnologías utilizadas

Enumera todas las tecnologías, herramientas y servicios que ya fueron elegidos para el proyecto, así como la razón de cada elección.

## 6. Decisiones de diseño

Resume todas las decisiones arquitectónicas importantes que se hayan tomado durante el proyecto, incluyendo aquellas que descartaron otras alternativas.

## 7. Estado actual del desarrollo

Describe exactamente qué partes ya están terminadas, cuáles están en proceso y cuáles aún no se han comenzado.

## 8. Último Issue trabajado

Indica:

* Nombre del Issue.
* Objetivo.
* Trabajo realizado.
* Si quedó terminado o pendiente.

## 9. Próximo paso recomendado

Explica cuál es el siguiente paso lógico para continuar el proyecto y por qué.

## 10. Estructura actual del repositorio

Muestra el árbol de directorios actualizado del proyecto.

## 11. Dependencias instaladas

Lista las librerías que actualmente forman parte del proyecto.

## 12. Configuración importante

Incluye cualquier configuración, variables de entorno, convenciones o detalles que el siguiente chat deba conocer.

## 13. Pendientes importantes

Enumera todas las tareas importantes que aún faltan por realizar.

## 14. Riesgos o decisiones abiertas

Menciona cualquier tema que todavía no esté decidido y que deba resolverse más adelante.

## 15. Contexto para el siguiente chat

Escribe un breve mensaje dirigido al siguiente ChatGPT explicando exactamente en qué punto del proyecto nos encontramos y desde dónde debe continuar.

---

Instrucciones adicionales:

* No inventes información.
* Usa únicamente información que ya se haya definido durante nuestras conversaciones.
* Si algo todavía no existe, indícalo explícitamente.
* Mantén el documento actualizado con el estado real del proyecto.
* No omitas decisiones importantes aunque parezcan pequeñas.
* El documento debe poder leerse por sí solo, sin necesidad de revisar conversaciones anteriores.
* Piensa que el siguiente chat no conoce absolutamente nada del proyecto, por lo que este documento debe contener todo el contexto necesario para continuar el desarrollo sin perder continuidad.
