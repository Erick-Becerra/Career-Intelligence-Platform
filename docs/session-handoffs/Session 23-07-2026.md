# Estado del Proyecto

## 1. Resumen del proyecto

El proyecto consiste en desarrollar un sistema de Inteligencia Artificial que automatice gran parte del proceso de búsqueda de empleo. El objetivo es construir una aplicación modular basada en múltiples agentes especializados, donde cada agente sea responsable de una tarea específica (buscar vacantes, filtrarlas, adaptar el CV, preparar aplicaciones y monitorear resultados).

El proyecto está siendo desarrollado como un proyecto personal y de aprendizaje, priorizando una arquitectura limpia, escalable y bien documentada.

---

## 2. Objetivo final

Construir una aplicación en Python capaz de:

* Buscar automáticamente ofertas de trabajo en diferentes plataformas.
* Filtrar las vacantes según el perfil del usuario.
* Calcular un porcentaje de compatibilidad entre el candidato y cada vacante.
* Adaptar automáticamente el CV para cada oferta utilizando un modelo de IA.
* Preparar la información necesaria para que el usuario únicamente revise y envíe la solicitud.
* Monitorear las vacantes ya procesadas y mantener un historial.

El usuario seguirá siendo quien envíe manualmente las aplicaciones. El sistema no realizará postulaciones automáticas.

Actualmente la arquitectura definida es una arquitectura modular en Python.

```text

├── docs/
│   |──session-handoffs/

│   |──Architecture.md

│   ├── utils/
│   ├── workflows/

│

├── README.md
```

### Responsabilidad de cada carpeta

**agents/**
Contendrá los agentes inteligentes del sistema.

**core/**
Configuración general del proyecto.

**services/**
Conexión con APIs y servicios externos.

**models/**
Modelos de datos del proyecto.

**utils/**
Funciones auxiliares reutilizables.

**workflows/**
Lógica que coordinará la ejecución entre agentes.

**main.py**
Punto de entrada de la aplicación.

Actualmente la mayoría de las carpetas aún están vacías.

---

## 6. Decisiones de diseño

Durante la planeación se tomaron las siguientes decisiones importantes:

### La aplicación será desarrollada principalmente en Python.

Inicialmente se consideró desarrollar gran parte del proyecto en n8n, pero posteriormente se decidió que Python sería el núcleo del sistema.

---

### Los agentes serán desarrollados manualmente.

No se utilizarán frameworks especializados como CrewAI o LangGraph durante la primera versión.

Cada agente será implementado como código propio para comprender mejor la arquitectura y evitar dependencias innecesarias.

---

### n8n no será el cerebro del proyecto.

Su función será únicamente automatizar tareas externas.

---

### Arquitectura modular.

Cada agente tendrá una responsabilidad claramente definida.

---

### Desarrollo incremental.

No se crearán carpetas, archivos o configuraciones que todavía no sean necesarias.

La estructura crecerá conforme avance el proyecto.

---

### El usuario seguirá teniendo el control de las aplicaciones.

El sistema no enviará solicitudes de empleo automáticamente.
